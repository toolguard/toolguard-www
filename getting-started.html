Create your account

ToolGuard stores your service credentials and audit logs securely in the cloud — your account is what keeps everything together.

Head to app.toolguard.ai and sign up. You can register with:

  • Google — one click, no new passwords to remember
  • GitHub — ideal if you already have a developer account
  • Email and password — works for everyone

The free plan includes everything you need to get started. No credit card required.

✅ Once you sign in for the first time, ToolGuard's built-in onboarding wizard walks you through the next steps automatically. You can also follow this guide manually at your own pace.
2

Connect Gmail

Connecting Gmail lets your AI read and search your inbox — without ever handing your password or OAuth token to the AI itself. ToolGuard acts as the secure intermediary.

In the ToolGuard dashboard, click Services → Connect Gmail. A standard Google authorization screen will appear. Click Allow to grant ToolGuard access to your Gmail.

  • Read-only by default — ToolGuard cannot send email on your behalf unless you explicitly enable it
  • Encrypted credentials — your Google token is stored encrypted; ToolGuard never sees your password
  • Revocable any time — remove access instantly from your Google account settings or the ToolGuard dashboard
ℹ️ If you're using the onboarding wizard, this step is handled automatically on first login — just follow the prompts on screen.
3

Install the ToolGuard connector

The connector is a small program that runs on your computer. It bridges your AI tools and the ToolGuard cloud — so your AI can make secure tool calls without credentials ever passing through it.

Pick your platform:

# Install via Homebrew (recommended)
brew install toolguard/tap/toolguard
# Download the latest release for Linux (amd64)
curl -L https://github.com/toolguard/toolguard/releases/latest/download/toolguard-linux-amd64 \
  -o /usr/local/bin/toolguard
chmod +x /usr/local/bin/toolguard

Or grab the binary directly from GitHub Releases and place it on your PATH.

Download the Windows executable from GitHub Releases. Grab the toolguard-windows-amd64.exe file, rename it to toolguard.exe, and add it to a folder on your PATH — or run it from wherever you saved it.

Once installed, run this command to log in and configure the connector automatically:

toolguard connect

It will open a browser window to complete authentication, then configure itself automatically. You're done with this step when the terminal shows a confirmation message.

4

Point your AI at the connector

Your AI needs to know where to find ToolGuard's MCP server. This one-time config step tells Claude, Cursor, or any other AI agent to route tool calls through ToolGuard.

The connector runs a local MCP server at http://localhost:6789 (or wherever you configured it). Add it to your AI tool's MCP server list:

// Add to ~/Library/Application Support/Claude/claude_desktop_config.json
{
  "mcpServers": {
    "toolguard": {
      "command": "toolguard",
      "args": ["start"]
    }
  }
}

Restart Claude Desktop after saving. You'll see "toolguard" listed in the MCP servers panel.

In Cursor, open Settings → MCP → Add Server. Set the transport to stdio and the command to toolguard mcp. Save and restart Cursor. 

# Or add to ~/.cursor/mcp.json
{
  "mcpServers": {
    "toolguard": {
      "command": "toolguard",
      "args": ["start"]
    }
  }
}

Any MCP-compatible agent can connect to ToolGuard using either the stdio or HTTP transport:

  • stdio transport: run toolguard mcp as a subprocess from your agent config
  • HTTP transport: point your agent at http://localhost:6789 (the connector must be running)

See the documentation for full configuration options and environment variables.

5

Test it — ask your AI about your email

A quick test confirms the full chain is working: your AI → ToolGuard connector → ToolGuard gateway → Gmail. If you get a real email back, you're all set.

In Claude Desktop, Cursor, or whichever AI you configured, type:

What's the latest email in my inbox?

If your AI responds with a real email subject, sender, and date — you're done! ToolGuard is routing the request securely in the background.

💡 Try asking for emails by topic, sender, or date range. ToolGuard's Gmail plugin supports search, read, and thread listing — all logged to your audit trail in the dashboard.

If something doesn't work, check the troubleshooting guide or reach out to support@toolguard.ai.
