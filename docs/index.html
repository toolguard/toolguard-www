üöÄ Quick Start

Get ToolGuard up and running in minutes with these simple steps.

Installation

# Install from npm
npm install -g toolguard

# Or install from source
git clone https://github.com/toolguard/toolguard.git
cd toolguard
npm install
npm run build

Configuration

Create a toolguard.config.json file in your project directory:

{
  "servers": {
    "filesystem": {
      "command": "npx",
      "args": ["@modelcontextprotocol/server-filesystem", "/path/to/allowed/directory"],
      "env": {}
    }
  },
  "security": {
    "allowedActions": ["read", "write"],
    "credentialIsolation": true,
    "auditLogging": true
  },
  "proxy": {
    "port": 3000,
    "host": "localhost"
  }
}

Running ToolGuard

Start the ToolGuard proxy server:

# Start with default config
toolguard proxy

# Or specify a custom config file
toolguard proxy --config /path/to/your/config.json

# Run with debug logging
toolguard proxy --verbose

Your MCP servers are now secured behind ToolGuard. Connect your AI agents to http://localhost:3000 instead of directly to the MCP servers.

üèóÔ∏è Architecture Overview

ToolGuard implements a security-first architecture that sits between AI agents and MCP servers, providing multiple layers of protection.

Core Components

  • Proxy Server: The main entry point that intercepts all MCP requests
  • Security Engine: Analyzes requests for potential security threats
  • Credential Manager: Isolates and manages sensitive credentials
  • Audit Logger: Records all activities for compliance and monitoring
  • Policy Engine: Enforces custom security rules and policies

Request Flow

  1. AI Agent sends request to ToolGuard proxy
  2. Authentication validates the agent's identity
  3. Policy Check ensures request complies with security rules
  4. Threat Analysis scans for potential security risks
  5. Credential Injection adds necessary credentials securely
  6. MCP Server receives the sanitized request
  7. Response Processing filters and validates the response
  8. Audit Logging records the complete transaction

Security Features

  • Credential Isolation: Credentials never exposed to AI agents
  • Request Sanitization: Malicious payloads filtered out
  • Policy Enforcement: Custom rules for allowed operations
  • Anomaly Detection: ML-powered threat detection
  • Audit Trail: Complete activity logging
  • Rate Limiting: Prevents abuse and DoS attacks

‚öôÔ∏è CLI Reference

ToolGuard provides three main commands to manage your security proxy:

proxy

Starts the main ToolGuard proxy server.

toolguard proxy [options]

Options:
  --config, -c     Path to configuration file (default: ./toolguard.config.json)
  --port, -p       Port to listen on (default: 3000)
  --host, -h       Host to bind to (default: localhost)
  --verbose, -v    Enable verbose logging
  --daemon, -d     Run as daemon process
  --pid-file       Path to PID file (daemon mode)

Examples:
  toolguard proxy
  toolguard proxy --config prod.config.json --port 8080
  toolguard proxy --daemon --pid-file /var/run/toolguard.pid

shim

Creates a transparent shim for existing MCP installations.

toolguard shim [options] <mcp-server-command>

Options:
  --config, -c     Path to configuration file
  --output, -o     Output shim script path
  --template, -t   Shim template to use (bash, powershell, docker)

Examples:
  toolguard shim "npx @modelcontextprotocol/server-filesystem /data"
  toolguard shim --template docker "mcp-server --port 3001"
  toolguard shim --output ./mcp-secure.sh "python mcp-server.py"

poller

Monitors MCP servers and ToolGuard health status.

toolguard poller [options]

Options:
  --config, -c     Path to configuration file
  --interval, -i   Polling interval in seconds (default: 60)
  --output, -o     Output format (json, text, prometheus)
  --webhook        Webhook URL for alerts
  --threshold      Alert threshold for response times (ms)

Examples:
  toolguard poller --interval 30
  toolguard poller --output prometheus --webhook https://alerts.example.com
  toolguard poller --threshold 1000 --output json

Global Options

These options are available for all commands:

Global Options:
  --help           Show help information
  --version        Display version information
  --quiet, -q      Suppress non-error output
  --log-file       Path to log file
  --log-level      Log level (error, warn, info, debug)
